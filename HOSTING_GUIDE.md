я# Инструкция по деплою (развертыванию) на SprintHost

## Шаг 1: Сборка проекта (Build)
В вашем терминале (PowerShell) выполните команду:
```powershell
npm run build
```
Эта команда создаст папку **`dist`** в корне проекта. В ней будут лежать готовые файлы сайта (оптимизированный HTML, CSS и JS), которые понимает сервер.

## Шаг 2: Загрузка файлов
1. Зайдите в панель управления **SprintHost**.
2. В левом меню выберите иконку папки — **«Файловый менеджер»**.
3. Перейдите по пути: `domains` -> `ваш_домен` (например, `bestfriend.ru`) -> `public_html`.
4. Удалите стандартные файлы (например, `index.php`), если они там есть.
5. Загрузите в эту папку **все содержимое** вашей локальной папки `dist` (папки `assets`, файл `index.html` и другие).

## Шаг 3: Настройка .htaccess (обязательно для React)
Чтобы при обновлении страницы или переходе по прямым ссылкам сайт не выдавал ошибку 404, нужно настроить перенаправление на `index.html`.

1. В «Файловом менеджере» в папке `public_html` создайте новый файл с именем **`.htaccess`**.
2. Вставьте в него следующий код:

```apache
<IfModule mod_rewrite.c>
  RewriteEngine On
  RewriteBase /
  RewriteRule ^index\.html$ - [L]
  RewriteCond %{REQUEST_FILENAME} !-f
  RewriteCond %{REQUEST_FILENAME} !-d
  RewriteRule . /index.html [L]
</IfModule>
```

---
**Совет:** Если вы планируете часто обновлять сайт, лучше настроить загрузку через FTP-клиент (например, FileZilla), данные для доступа к которому можно найти в письме от SprintHost или в разделе «FTP» панели управления.
