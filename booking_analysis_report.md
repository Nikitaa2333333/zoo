# Анализ логики бронирования сайтов Scandicat и Bonifacy Holidays

## 1. Общие выводы и техническая база

Оба проанализированных сайта (scandicat.ru и bonifacyholidays.ru) используют **одну и ту же стороннюю облачную систему управления гостиницами (PMS) — [LitePMS](https://litepms.ru/)**. 

### Как реализована интеграция (Анализ кода страницы)
В обоих случаях бронирование реализовано по модели **Embedded Widget** (встроенный виджет). 
В коде сайтов нет сложной самописной логики бэкенда для бронирования. Вместо этого в код вставлен небольшой JavaScript-сниппет:

1. На странице инициализируется переменная `litepmsembed_id` (уникальный ID гостиницы в системе LitePMS).
2. Подгружается внешний скрипт `https://litepms.ru/js/widget_embed.js`.
3. Этот скрипт динамически создает `iframe` (окно), который подгружает форму поиска и бронирования прямо с серверов LitePMS.
4. Вся сложная логика — расчет цен, проверка доступных номеров по датам, применение скидок и сбор данных клиента — происходит **на стороне LitePMS**, а не самого сайта.

## 2. Разбор логики бронирования: Scandicat
**Ссылка:** https://scandicat.ru/бронирование/
**Ключевая фича (то, что вам понравилось):** Бронирование с возможностью выбора количества питомцев.

### Сценарий бронирования:
1. **Экран поиска:** Пользователь выбирает даты заезда и выезда, а также указывает количество питомцев (от 1 до 4) в выпадающем списке.
2. **Экран выбора номера:** Система показывает доступные категории номеров (Комфорт, Премиум, Люкс и т.д.) с итоговой ценой за весь период.
3. **Оформление:** При клике "Забронировать" открывается форма ввода данных (ФИО, телефон, e-mail, время заезда).

### Как работает логика выбора кол-ва питомцев (ценообразование LitePMS):
* Базовая стоимость номера в системе задана из расчета на **одного** питомца.
* В настройках тарифа LitePMS включена опция "дополнительное место".
* Если пользователь выбирает 2 и более питомцев, система автоматически добавляет фиксированную доплату (например, **+100 руб./день** за каждого дополнительного котика, если размер вольера это позволяет).
* *Пример:* Номер стоит 1200 руб./день за 1 кота. При выборе 2 котов тариф пересчитывается "на лету" и становится 1300 руб./день.

## 3. Разбор логики бронирования: Каникулы Бонифация (Bonifacy Holidays)
**Ссылки:** https://bonifacyholidays.ru/reservation.html, https://bonifacyholidays.ru/akcii.html
**Ключевая фича (то, что вам понравилось):** Акционные скидки в зависимости от срока проживания.

### Сценарий бронирования и логика скидок:
Сценарий шагов в виджете точно такой же (поиск -> выбор -> ввод данных), но используется более сложная логика автоматических скидок.

1. **Динамический пересчет стоимости (Скидка от срока):**
   Система LitePMS позволяет настраивать правила (акции/спецпредложения), которые автоматически изменяют базовый тариф в зависимости от длительности периода бронирования.
   * **Проживание от 11 до 29 суток:** Автоматически применяется скидка **10%** на общую сумму.
   * **Проживание от 30 суток:** Автоматически применяется скидка **20%**.
   * *Пример:* Выбираем даты с 21 марта по 5 апреля (15 дней). Если базовая цена 750 руб/день, система тут же показывает перечеркнутую старую цену и применяет скидку 10%, выставляя 675 руб/день.

2. **Работа с URL-параметрами:**
   Сайт отлично реализовал передачу параметров. Если вы кликаете по ссылке вида `#date_in=21-03-2026&date_out=05-04-2026`, виджет "ловит" эти параметры из адресной строки и сразу показывает результаты конкретно для этих дат, минуя первый экран поиска.

3. **Накопительные скидки (программа лояльности на странице акций):**
   На странице "Акции" указано, что есть накопительная скидка (10% с 5-го бронирования, 15% с 10-го). В базовом виджете без личного кабинета это автоматически *не применяется*. Такие скидки обычно пересчитываются администратором зоогостиницы вручную после оформления заявки в панели управления LitePMS, когда они видят карточку постоянного клиента.

## Итог: Как реализовать себе такой же функционал?

Если вы хотите достичь точно такого же UX (пользовательского опыта) и логики работы, у вас есть **два пути**:

### Путь 1: Использовать готовое SaaS решение (как сделали они)
Самый быстрый и дешевый вариант — зарегистрироваться в PMS для гостиниц (например, LitePMS, Bnovo, TravelLine, EasyWeek), настроить там шахматку номеров, тарифы, наценку за питомцев и скидки за срок проживания. После этого просто вставить `iframe` виджет на свой крутой фронтенд. 

* **Плюсы:** Не нужно писать сложный бэкенд, логику платежей и проверку доступности (овербукинг). Скидки и цены за питомцев настраиваются галочками в панели администратора.
* **Минусы:** Дизайн виджета стандартизирован, его нельзя кардинально поменять (можно только настроить цвета кнопок и шрифты).

### Путь 2: Написать свой Custom Backend + Frontend (Свой "booking-engine")
Если вы хотите, чтобы весь процесс происходил бесшовно прямо в вашем UI (на React/Вью и т.д.):
1. **База данных:** Таблицы `Rooms`, `Bookings`, `Rates`, `Discounts`.
2. **Логика корзины (Скидка от срока):** Алгоритм на бэкенде: `if (days >= 11 && days <= 29) { price = basePrice * 0.9; } else if (days >= 30) { price = basePrice * 0.8; }`.
3. **Логика доп. мест (Кол-во питомцев):** Базовая цена `basePrice` + `(petCount - 1) * extraPetSurplus`.
4. **Фронтенд:** Красивые React/Vue компоненты с календарями, где при изменении дат React отправляет запрос на бэкенд для "preview" суммы, и она мгновенно обновляется с красивой анимацией (с перечеркиванием старой цены при достижении 11-го дня).
